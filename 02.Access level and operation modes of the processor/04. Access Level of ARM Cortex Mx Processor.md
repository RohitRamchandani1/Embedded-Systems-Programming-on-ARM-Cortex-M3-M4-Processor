# Access Level of the processor
1. Processor offers 2 access levels
  - Privileged Access Level (PAL)
  - Non Privileged Access Level (NPAL)

2. If the code is running with Privileged Access Level (PAL), then it has full access to all the processor specific resources and restricted registers.

3. By `default` the code always runs in `Privileged Access Level` and `Thread Mode`.

4. If the code is running with Non-Privileged Access Level (NPAL), then it may not have access to some of the restricted registers of the processor.

5. When the processor is in `Thread Mode` it is possible to move the processor in NPAL. Once we move it out of the Privileged Access Level (PAL) to Non-Privileged Access Level (NPAL) then it is not possible to come back to PAL unless the processor’s operational mode is changed to `Handler Mode`.

6. When the processor is in the `Handler Mode`, execution is always in Privileged Access Level(PAL).

7. We can change the access level of the processor using the control register, if we want to switch the access level of the processor.
